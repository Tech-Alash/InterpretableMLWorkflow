Загрузка и предобработка данных
========================================================

Это самая близкая к стилю ``data_loading`` страница, адаптированная под данный проект.
Здесь показано, как репозиторий превращает исходный CSV FRED-MD в преобразованную таблицу, которую потом использует forecasting pipeline.

Общий поток обработки
========================================

Логика начальной загрузки данных находится в :func:`experiment.prepare_raw_data`, а переиспользуемые функции преобразования лежат в :mod:`helpers.utils`.

Последовательность предобработки такая:

1. загрузить ``data/2021-08.csv``;
2. считать коды преобразований, записанные в первой строке файла;
3. удалить строку с кодами и преобразовать ``sasdate`` в ``date``;
4. оставить только переменные с минимальным количеством пропусков;
5. удалить строки с missing values;
6. очистить имена колонок от спецсимволов;
7. позже, внутри :func:`helpers.utils.transform_data`, применить разности, процентные изменения, логарифмы и лаги.

Почему важна первая строка файла
==================================================

В файле FRED-MD в первой строке лежат подсказки о преобразованиях переменных.
Код превращает эту строку в Python-словарь:

* ключи: исходные имена переменных;
* значения: коды преобразований McCracken and Ng.

Позже эти коды комбинируются с горизонтами вроде ``change_predictor`` и ``change_target``.

Специфичные для проекта правила очистки
=================================================================

Загрузчик применяет несколько решений, которые легко пропустить при быстром просмотре кода:

* Переменные фильтруются до подмножества с минимальным количеством пропусков.
  Это позволяет получить плотную modelling-table без ручной импутации.
* Из имён признаков удаляются спецсимволы, потому что часть моделей плохо работает с исходными FRED-style названиями.
* Целевая переменная позже копируется в колонку с буквальным именем ``target`` внутри :func:`helpers.utils_predict.run_experiments`.

Базовый набор переменных из статьи
=================================================

В paper авторы подчёркивают, что лучший баланс между качеством и интерпретируемостью
часто достигается не на максимально широком наборе переменных, а на компактном
наборе признаков, выбранных экспертно.

Для baseline-анализа использовались макроэкономические ряды, покрывающие разные каналы:

* безработица;
* краткосрочная процентная ставка;
* личные доходы;
* промышленное производство;
* потребление;
* фондовый рынок;
* бизнес-кредиты;
* инфляция;
* нефть и денежная масса.

В коде эта идея отражена через ``features_key`` в :mod:`helpers.config`.
Это важный момент интерпретации:

* маленький набор признаков упрощает объяснение модели;
* доменная экспертиза используется не после модели, а уже на этапе постановки;
* благодаря этому outputs легче обсуждать в прикладном контексте.

Основная функция преобразования
=================================================

Основная работа делается в :func:`helpers.utils.transform_data`.
Эта функция:

* делает колонку даты индексом;
* применяет код преобразования ко всем рядам, перечисленным в ``transformations``;
* создаёт лагированные предикторы;
* обрезает выборку по заданным начальной и конечной датам;
* при необходимости удаляет неполные строки.

Коды преобразований
========================================

:func:`helpers.utils.variable_transformer` поддерживает такие шаблоны:

.. list-table::
   :header-rows: 1

   * - Префикс кода
     - Значение
   * - ``1``
     - Без преобразования.
   * - ``2``
     - Первая разность.
   * - ``3``
     - Вторая разность.
   * - ``4``
     - Логарифм уровня.
   * - ``5``
     - Первая логарифмическая разность.
   * - ``6``
     - Вторая логарифмическая разность.
   * - ``7``
     - Дельта процентного изменения.
   * - ``pc``
     - Процентное изменение.
   * - ``lag``
     - Лагированное значение.

Итоговые строки преобразований состоят из кода и горизонта, например ``"5-12"``.

Куда дальше идут преобразованные данные
===============================================================

Преобразованная modelling-table не записывается на диск напрямую.
Вместо этого она передаётся в :func:`helpers.utils_predict.run_experiments`, где уже режется на rolling или expanding training windows и test segments.
