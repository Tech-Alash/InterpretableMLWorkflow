Настройка экспериментов
========================================

Матрица экспериментов создаётся модулем :mod:`setup_experiments`.
Каждый параметр в словаре конфигурации может быть:

* одиночным значением, которое остаётся фиксированным для всех запусков;
* списком значений, из которого строятся несколько комбинаций экспериментов.

Источник конфигурации по умолчанию
==================================================

Конфигурация по умолчанию задаётся прямо в коде:

Справка по параметрам
========================================

Ниже перечислены параметры, которые сильнее всего влияют на эксперимент.

.. list-table::
   :header-rows: 1
   :widths: 25 20 55

   * - Параметр
     - Пример
     - Смысл
   * - ``method``
     - ``["Ridge", "LightGBM"]``
     - Семейства моделей для сравнения. Каждое значение добавляет новые строки в матрицу экспериментов.
   * - ``features``
     - ``["key"]``
     - Выбор набора признаков. Обычно используются ключевые переменные, все переменные или PCA-варианты.
   * - ``target``
     - ``"UNRATE"``
     - Целевая переменная, которую нужно прогнозировать.
   * - ``lag``
     - ``12``
     - Глубина лага предикторов при построении признаков.
   * - ``change_predictor``
     - ``3``
     - Горизонт, используемый при преобразовании предикторов.
   * - ``change_target``
     - ``12``
     - Горизонт, используемый при преобразовании target.
   * - ``lag_response_as_predictor``
     - ``["response"]``
     - Управляет тем, как преобразуется лаг target, если он снова используется как предиктор.
   * - ``step_size``
     - ``12``
     - Как часто модель переобучается при движении рекурсивного forecasting loop.
   * - ``window_size``
     - ``10000``
     - Максимальная длина training window. Меньшее значение фактически создаёт rolling-window режим.
   * - ``winsorize``
     - ``[0.01]``
     - Уровень отсечения хвостов для training и test predictors.
   * - ``norm_var``
     - ``true``
     - Нужно ли стандартизовать предикторы перед обучением и оценкой.
   * - ``hyper_type``
     - ``["kfold_block_gap"]``
     - Схема cross-validation для hyperparameter search.
   * - ``n_folds``
     - ``5``
     - Количество folds в кросс-валидации.
   * - ``cv_repeat``
     - ``1``
     - Число повторов для повторяемых вариантов cross-validation.
   * - ``cv_mode``
     - ``3``
     - Частота, с которой во времени обновляется hyperparameter search.
   * - ``random_search_iterations``
     - ``100``
     - Сколько случайных комбинаций параметров проверять в randomized search.
   * - ``bootstrap_hyper``
     - ``false``
     - Выполнять ли сам hyperparameter tuning на bootstrap-sample.
   * - ``n_boot``
     - ``[30]``
     - Число bootstrap-trained моделей на один эксперимент.
   * - ``n_boot_importance``
     - ``[30]``
     - Число bootstrap-итераций, используемых при расчёте важности признаков.
   * - ``bootstrap_proportion``
     - ``1``
     - Относительный размер каждой bootstrap-выборки.
   * - ``permutation_rep``
     - ``250``
     - Количество повторений на каждый признак при расчёте permutation importance.
   * - ``seed``
     - ``[null]``
     - Управление random-state для sampling гиперпараметров и распределения по folds.
   * - ``counter``
     - ``[0]``
     - Идентификатор повтора эксперимента, который тоже входит в hash.

Как разворачиваются комбинации
=================================================

:func:`helpers.utils.model_val_params` и :func:`helpers.utils.list_experiments` перебирают все комбинации параметров, заданных списками.

Это означает, что:

* два значения в ``method`` и
* три значения в ``counter``

уже дадут шесть строк экспериментов ещё до разворачивания остальных списков.

Удаление дубликатов
========================================

Некоторые комбинации по сути дублируют друг друга.
Например, настройка hyperparameter-search может не иметь смысла для модели без настраиваемых гиперпараметров.

Такое упрощение выполняется в :func:`helpers.utils.remove_duplicates`.

Практические советы
========================================

Когда меняешь ``setup_files/main.yaml``:

* меняй по одной оси за раз, если хочешь контролируемый рост количества запусков;
* внимательно смотри на ``n_boot``, ``n_boot_importance`` и ``permutation_rep``, потому что они сильно увеличивают время расчёта;
* помни, что каждый лишний элемент списка может развернуться во множество дополнительных hash-идентифицированных запусков.

Базовая постановка из paper
---------------------------

Если твоя цель - быть максимально близко к исследованию, полезно понимать,
какие элементы baseline-сценария были центральными в статье.

По содержанию paper базовая постановка включает:

* прогноз изменения безработицы в США;
* горизонт прогноза 12 месяцев;
* monthly macroeconomic data из FRED-MD;
* небольшой экспертно подобранный набор предикторов, покрывающих разные макроэкономические каналы;
* expanding-window pseudo out-of-sample evaluation;
* абсолютную ошибку как основную performance metric.

Важно понимать, что репозиторий позволяет варьировать постановку шире, чем это было сделано в baseline.
То есть YAML-конфигурация служит сразу двум целям:

* воспроизведение центрального сценария статьи;
* запуск robustness-checks и альтернативных постановок.

Практически это означает следующее:

* если тебе нужно воспроизвести paper как можно ближе, меняй конфигурацию минимально;
* если тебе нужно исследовать чувствительность результатов, меняй один блок параметров за раз и сохраняй связь с baseline.

Какие параметры особенно влияют на смысл эксперимента
-----------------------------------------------------

Не все параметры одинаково важны с содержательной точки зрения.

Сильнее всего меняют интерпретацию результата:

* ``target`` и горизонты ``change_target`` / ``change_predictor``;
* состав признаков через ``features``;
* список моделей в ``method``;
* схема оценки во времени через ``step_size`` и ``window_size``;
* объём bootstrap и importance-оценок через ``n_boot`` и ``n_boot_importance``.

Именно эти параметры в первую очередь определяют, насколько твой запуск остаётся
близким к paper или уже становится отдельным исследовательским вариантом.
